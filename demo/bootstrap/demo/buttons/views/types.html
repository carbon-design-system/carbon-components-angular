Button types

There are two main button types: simple and menu.

Use simple or toggle buttons when the visibility of actions is important. Actions that are contained in a menu button (drop-down or combination) are less visible because the user needs to search through the menu to discover them, so use menu buttons when real estate is a constraint.

Simple buttons

Simple buttons result in an immediate action. Clicking/pressing the button immediately performs the action specified by the label.

See the button hierarchy section for detailed descriptions of the various simple buttons and when to use them.

Drop-down menu buttons

Drop-down menu buttons are used to combine multiple actions into a single button. They are used when you need to present a set of actions and the real estate is constrained. Clicking a drop-down menu button drops down a list of actions.

Combination (split) menu buttons

Combination buttons are specialized versions of drop-down menu buttons that assign a default action to the button label. Other related actions are provided via the drop-down list.

Clicking the left portion of the button immediately performs the action specified by the label. Clicking the arrow drops down a menu of related actions. The action associated with the button label should also be included in the menu. The label must always reflect the action associated with the button so that the user knows what it will do.

A combination button can be static or dynamic. The action associated with a static combination button is always the same (for example, the default or commonly used action). The action associated with a dynamic combination button changes based on the selection in the menu (for example, the last action). Dynamic combination buttons are best used for repetitive tasks where the action varies from one user to another.
