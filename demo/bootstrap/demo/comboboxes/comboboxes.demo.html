Combination (a.k.a. Combo) Boxes

Users need to choose one or more items from a list, and they need to be able to add new items to the list as well.

The combination box element is still in progress.

What they're for...

A combo box is a way to show a list of items, as well as to add an item. A combo box is a combination of a drop-down list, or a list box, and a single line text field.

Use when...

Use a combo box when the user needs to select one or more items from a list, and where they can add a new item to the list as well.

Implementation

A combo box is a combination of a drop-down list or a list box, and a single line text field that allows the user to type a value that isn't in the list, or to find and choose from the existing list.

An editable drop down list is a combination of a drop-down list and an editable text field. This combination is most commonly used, to save real estate.

An editable list box is a combination of a list box and an editable text field. This combination is less commonly used because it uses so much of the space in the window. The combination of list box and text field is used to encourage the user to review the alternative choices, and draw attention to an item in the list.

  • If space is constrained, implement the combo box with a drop-down list instead of a list box. This combination is most commonly used, to save real estate.
  • Implement the combo boxes with a list box when space is unconstrained. This combination is less commonly used because it uses so much of the space in the window. The combination of list box and text field is used to encourage the user to review the alternative choices, and draw attention to an item in the list.
  • If a user enters text that is not valid (for example, text instead of a number) you must handle the error with an error message.
  • Clicking the arrow, or using the Alt+Up/Down arrow or Up/Down arrow keys, opens the drop-down list so the user can see likely choices in advance, and a list of previous user values.
  • Ideally, the list of possible values does not exceed 30. If there are more than 30 choices, consider showing a representative sample. This works well when the user can easily enter a different value. (For example, you can list the most common font sizes, and users who want to use a less common size can type it in.) However, if a user might have difficulty entering the value correctly, it is better to display the full list of choices and scroll the list if needed.

Order of choices

Unless a more natural order is appropriate, sort the list items alphabetically.

Default choices

  • When there is a reasonable default value, supply it initially in the text field and as selected in the list.
  • When there is a single click in the text field, or when the text field receives focus, the text in the entry should automatically be selected.
  • With the first character entered, the default or previous value is automatically removed.
  • When there is no default value, you may use a prompt label in the text field for identifying the purpose of the combo box.
  • It must not be crucial information that user needs to see while using the combo box.
  • The prompt text should be in italic and in grey. It must not be confused with real text in black. Use sentence style capitalization for the prompt text.

Alignment

  • The text in a combo box does not wrap. Use clear and concise items.
  • Avoid using horizontal scroll. If necessary, resize the list, or display the tail and the beginning of the value instead.
  • Sometimes expanding the drop-down list overlaps other UI elements.

States

A combo box can have these states:

Enabled
The user can choose from the list, or create a new entry.
Disabled
Both the disabled combo box and the label are unavailable to the user. The combo box cannot be selected. If an option is preselected, it is still visible in gray text.

Interactions

  • When the user selects, clicks or moves keyboard focus to the text field, the cursor is set and the user can type a value. The items are mutually exclusive.
  • Clicking the arrow of the drop-down list or using the Alt+Up/Down arrow keys, opens the drop-down part, but does not place the focus on the text field.
  • Text does not wrap in this control. Write the list choices as concisely as possible.

Clicking on, touching or tabbing to the text field

  • When the user clicks on, touches, or tabs to the text field part of the combo box, focus is set with all the current text selected, so the user can start typing immediately to replace existing text.
  • For combo box with a drop-down list, as the user types, the drop-down list opens when there is a potential match in the list, showing the list of potential matches and highlighting the first match. The opened list closes if the user continues to type letters and there are no longer any potential matches. If there is never a match, the list does not open. If the list is always displayed, no highlight selector is shown until a match is found. The items in the drop-down filter use the same rules as auto-complete. Auto-complete involves the program predicting a word(s) that the user wants to type in without the user actually typing it in completely.
  • Until the user highlights a particular choice, only the typed letters are displayed in the text field.

Clicking or touching the drop down arrow, or pressing the Alt+Up/Down arrow or Up/Down arrow when focus is on the text field

  • The drop down list opens, so the user can see the options in the list.
  • The default or preselected list option is automatically highlighted.
  • Focus remains in the text field, so the user can type characters.
  • For desktop applications, additional key presses of Alt+Up/Down arrow keys toggles the drop-down list closed/opened.

Selecting an option by touching, clicking with the mouse, or pressing the Enter key

  • The drop-down list closes.
  • The selected option is shown in the text field part of the combo box.
  • Focus returns to the text field.

Exiting the drop down list by touching outside it, or pressing Esc key

  • The drop-down closes.
  • Focus returns to the text field

Keyboard (Desktop) Only Interactions

Pressing Up/Down arrow key when the drop down list is open

  • Places keyboard focus on the selector. If there was no selector, the selector and keyboard focus is placed on the first option in the list.
  • Moves the highlight selector up/down in the drop-down list.
  • Updates the contents of the text field to the text of the currently selected option.
  • When keyboard navigation is at the top/bottom element in the drop down list, Up/Down arrow keys do not cause the selector to wrap around to the bottom/top element. Instead the selector remains on the same element in the list.
  • Saves the current cursor position in the text field.

Pressing Home/End key when keyboard focus is in the text field

  • Removes selection highlight from text in the text field.
  • Moves the input cursor in front of the first character or after the last character, respectively.

Pressing Home/End key when keyboard focus is on an option in the list

  • Moves the highlight selector and keyboard focus to the first/last option in the drop-down list.

Pressing page Up/Down key when keyboard focus is in the list

  • Pages the content of the drop-down list up/down.
  • Moves the highlight selector to the topmost/bottommost visible list option.

Pressing Tab key when the keyboard focus is in the list

  • Updates the contents of the text field part of the combo box to the text of the currently selected option.
  • The drop-down list closes.
  • Moves keyboard focus to the next interactive UI element in the Tab order.

Typing characters when keyboard focus is on the list

  • Returns focus to the text field at the saved cursor position.
  • Other behaviors are documented in the first bullet above.

Accessibility

There are no standard HTML elements for a combo box. A combo box is the combined presentation of a single line text field with a drop-down list or a list box. Keyboard events must be built into the control so the user can interact with the combo box as specified in the usage section of this standard.

When larger fonts are used, the text in the text field and the selection choices should still be visible.

When hint text is provided in the text field that has no default or prior selection, ensure it has good color contrast. The contrast between text (and images of text) and the background behind the text must meet the following minimum requirements based on the size of the text:

  • Small text: 4.5:1 contrast ratio for plain text that is less than 18 point if not bold and bold text that is less than 14 point.
  • Large text: 3:1 contrast ratio for plain text that is at least 18 point if not bold and bold text that is at least 14 point.

The WAI-ARIA markup for the combo box is as follows:

  • The container element for the combo box has a role="combobox".
  • Elements with the role combobox have an implicit aria-haspopup="true".
  • The combo box must have aria-expanded="true" when the drop-down list is displayed or aria-expanded="false" when it is not.
  • Set the property aria-disabled="true" when the combo box is disabled and aria-disabled="false" when it is enabled.
  • The first child element within the combo box is an input text field with a role="textbox" and is responsible for managing the keyboard focus between the text field and the drop-down list as well as displaying the list. You must ensure this text field is in the tab order.
  • Set the aria-autocomplete attribute on the text field to indicate whether completion suggestions are provided and how the suggestions are provided. Depending on this attribute's value, the following must also be done to ensure interoperability with assistive technology:
    • When aria-autocomplete="none" (the default setting), authors must manage and set focus on the associated list box, so assistive technologies can follow the currently selected value.
    • When aria-autocomplete is set to inline or both, authors must update the value of the focused text field, so assistive technologies can announce the currently selected value.
    • When aria-autocomplete="list", user agents (browsers) or assistive technologies must follow the aria-activedescendant of the combo box, and authors should associate the combo box text field with its list box using aria-owns.
  • Whenever the list of options is a drop-down, the next element is an HTML <button>, or another element with a role="button". This element is used to toggle the display of the combo box's drop-down list and is not included in the tab order because keystrokes are available to open the drop-down list.
  • The next element has a role="listbox" and represents the drop-down list. It manages keyboard navigation among list items and navigating back to the text field if necessary.
  • Each item in the list box has a role="option". Options are not in the tab order.
  • Provide a label for the combo box by referencing the text field in the combo box. You can use an aria-label to associate a hidden label with the combo box or you may use the HTML <label> element with the "for" attribute to reference the text field.
  • To be keyboard accessible, authors need to manage the keyboard focus of descendants for all instances of the combo box role.
  • If the list box is resizable, it must be resizable using the keyboard, such as adding a Resize list option to the context menu for the list box. Note, context menus are always displayed using Shift+F10. In addition, you can add a tooltip to the expandable indicator icon that says "Use right/left arrow key to resize the list" and implement those keys to expand/contract the list.

Globalization

For translated text, use leading edge to specify the placement of the combo box label. Use sentence style capitalization for combo box label. Use sentence style capitalization for choices unless the item is a proper noun.

See the Globalization sections for drop-down lists, and list boxes for more information.
